Techniques and Data Sources We Used

Property Data Scraping from Zillow.com

We used the urllib2 module from python and requested the related rental information from Zillow.com by going through all compatible zipcodes within I-285 highway. We then encoded the webpage and collected the name, coordinates, address, type and link for each property.

Commuting Time Collection with Google Map Distance Matrix API

In order to obtain approximate commuting time from rental places to school, we utilized Google Map Distance Matrix API to request travel durations between two given locations. Also, we will consider three main modes of transportations which is driving, transiting and walking.

Life Convenience Evaluation with Yelp API

Data for convenience evaluation is downloaded from Yelp API. Every estate property is chosen as a searching center. Several category data has been involved to evaluate the convenience around the searching center. Besides, every estate property will be given a yelp_grade which represent the level of convenience for this property.

Property Data Cleaning with OpenRefine

Since data from Zillow.com does not have an uniform return for each request. The original data we scraped had many duplicates and blank tuples. So we utilized OpenRefine to get rid of all the dirty tuples. Finally the number of rental properties result we obtained is 1098, and all those properties are located within I-285 highway in Atlanta.

Local Crime Data Collection

Local crime data are downloaded from Atlanta Police Department and scraped from crimemapping.com. We use the population by zip code to normalize crime rate, and give a crime score of certain address with crime reports number divided by population within I-285 area.

Interactive Visualization

Introducing Google Maps JavaScript API, we build a web application enabling users easily interact with our database, by sending filters to Django and elegantly visualizing the data returned from backend with Ajax, jQuery, D3.js and so on.

Common Questions

What is GT rent?

Along with the increasing student population in Georgia Tech, those students who prefer living off campus may be confronted with an accommodation puzzle. Rents, distances to school, life convenience and crime rates are several major concerns in decision making. Thus we polish and integrate the data into a web application with the data visualization and analytics skills, which aims to provide best local rental information and recommendation for Georgia Tech students.

Why do you make this?

This project is inspired by our painful experiences in searching for suitable accommodations. Details about these rental properties are scattered throughout several major data sources such as Google Maps, Yelp, and rental housings’ official websites, making the searching process difficult. On the other hand, some existing tools like Zillow and Trulia are unnecessarily complicated for GT students. Thus, our project aims to create an application that combines all necessary features in a more concise and dedicated way.

What are your innovations?

  • Generating crime or convenient scores of properties lack of data with kNN algorithm.
  • Clustering all rental properties with K-means algorithm and providing the users with recommendation of similar properties.
  • Building the web app on AWS based cloud server Heroku with runtime data management.
